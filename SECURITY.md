# Security Policy

## Supported Versions

Only the latest release of Dartform receives security updates.

| Version | Supported |
|---|---|
| Latest | ✅ |
| Older versions | ❌ |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability in Dartform, please report it responsibly by emailing:

📧 **[security@dartform.dev](mailto:security@dartform.dev)**

Include in your report:
- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof of concept
- The Dartform version affected
- Your contact information (optional, for follow-up)

### What to expect

- We will acknowledge your report within **48 hours**
- We will provide a status update within **7 days**
- We will notify you when the vulnerability has been patched
- We will credit you in the release notes (unless you prefer to remain anonymous)

We appreciate responsible disclosure and will not take legal action against researchers who follow this policy.
